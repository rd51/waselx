Tool Recommendation

VS Code ✓ Recommended
Better for this project
• Multiple .py files stay open side-by-side
• Git integration — commit each phase as a checkpoint
• README.md editing with live preview
• Integrated terminal for running scripts
• Linting catches heap/tree bugs before you test
• Extension: Python + Pylance for autocomplete
Jupyter Notebook
Good only for specific parts
• Nice for Q3 (matrix display), Q22 (inline charts)
• Poor for class-based code (BST, AVL, Heap)
• Can't easily split into multiple source files
• Hard to turn into a deliverable .py file
• State management bugs in long notebooks
💡 Best of both: write all code in VS Code (.py files), but use a Jupyter notebook only for Q3 and Q22 to display matrices and charts inline. Submit .py files as the primary deliverable.

Critical Things to Keep in Mind

⚠ Academic Integrity: AI-generated code MUST be marked in comments. Unattributed AI code = violation.
⚠ No heapq/collections as primary: Build heap, BST, AVL from scratch. Only use stdlib for benchmarks.
⚠ Step-by-step workings required: Dijkstra hand trace, Floyd-Warshall matrices, AVL rotations — all must show intermediate steps, not just final answers.
⚠ NetworkX for visualization only: Never use it for algorithm computation.
⚠ Python 3.10+ — use match statements, type hints where helpful.
⚠ Include a README.md with setup instructions and sample run commands.
